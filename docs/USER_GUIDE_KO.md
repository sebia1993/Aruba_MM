# Aruba MM Cleanup 사용자 가이드

## 목적

`show global-user-table list role profiling` 조회 결과에서 사용자 MAC을 수집하고, 60초 대기 후 `aaa user delete mac <mac>`으로 삭제 처리합니다.

## 화면 구성

- `MM IP/Host`: 접속할 Aruba MM 주소입니다.
- `계정`, `암호`: SSH 접속 계정입니다.
- `Enable 암호`: enable 단계가 필요한 환경에서만 입력합니다.
- `Role`: 기본값은 `profiling`입니다.
- `주기(초)`: 자동 반복 실행 간격입니다. 최소 60초입니다.
- `이번 삭제 취소`: 조회 후 60초 대기 중 이번 실행의 삭제만 취소합니다.
- `세션 연결 해제`: 현재 유지 중인 MM 세션을 수동으로 종료합니다.

## 결과 확인

대시보드는 실행마다 아래 수치를 표시합니다.

- 조회: 이번 조회에서 발견한 MAC 수
- 삭제 성공: 삭제 명령이 성공한 MAC 수
- 삭제 실패: 오류가 발생한 MAC 수
- 남은 MAC: 삭제 후 재조회에서 남은 MAC 수

실행 요약은 결과 폴더의 `cleanup_summary.json`에 저장됩니다. raw 장비 출력은 저장하지 않습니다.

## 세션 동작

- GUI는 프로그램이 켜져 있는 동안 같은 MM 접속 정보를 재사용합니다.
- 수동 1회 실행과 주기 실행 모두 같은 세션으로 조회, 삭제, 삭제 후 검증 조회를 처리합니다.
- 접속 정보가 바뀌면 기존 세션을 닫고 새로 접속합니다.
- 프로그램을 완전히 종료하거나 `세션 연결 해제`를 누르면 세션을 닫습니다.
- 장시간 유휴 상태로 세션이 끊긴 경우 명령 실패 시 한 번 재접속하고 같은 명령을 재시도합니다.
- CLI는 한 번 실행할 때만 세션을 사용하고 실행 종료 시 연결을 닫습니다.

## 운영 주의

이 도구는 실제 사용자 세션을 삭제합니다. 처음 사용할 때는 운영 영향이 없는 시간대에 수동 1회 실행으로 조회 결과와 카운트다운 취소 동작을 먼저 확인하세요.

같은 MAC이 여러 줄에서 발견되어도 정규화된 MAC 기준으로 `aaa user delete mac <mac>` 명령은 한 번만 실행합니다.
